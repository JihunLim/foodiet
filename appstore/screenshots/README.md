# App Store Screenshots

## 폴더 구조

```
screenshots/
├── raw/                  # 원본 스크린샷 — 여기에 기기에서 찍은 사진을 드래그.
├── iphone_6_9/           # iPhone 6.9" (1290 × 2796) — 필수
├── iphone_6_5/           # iPhone 6.5" (1242 × 2688) — 호환, Apple 이 자동 생성해주기도 함
├── ipad_13/              # iPad 13" (2064 × 2752) — iPad 지원 시 필수
└── ipad_12_9/            # iPad 12.9" 3세대+ (2048 × 2732) — iPad 레거시
```

## 제출 요구사항 (2026-04 기준)

App Store Connect 는 다음 두 묶음을 업로드 받는다:

| 기기군 | 해상도 | 장수 | 상태 |
|---|---|---|---|
| iPhone 6.9" (iPhone 16 Pro Max급) | 1290 × 2796 | 3–10장 | **필수** |
| iPad 13" (iPad Pro 13") | 2064 × 2752 | 3–10장 | **iPad 지원 시 필수** |
| iPad 12.9" (Pro 3–6세대) | 2048 × 2732 | 3–10장 | 레거시 (13"가 있으면 생략 가능) |
| iPhone 6.5" (iPhone 11 Pro Max급) | 1242 × 2688 | 옵션 | 6.9"에서 자동 스케일되므로 생략 가능 |

## 캡처 가이드 (사용자)

**iPhone (실기기 권장)**
1. 기기에서 foodiet 앱을 연 상태로 아래 6장면을 각각 캡처 → `raw/` 폴더에 iCloud/AirDrop 으로 옮김.
2. 파일명은 `iphone_N_이름.PNG` 형식 권장 (예: `iphone_1_home.PNG`, `iphone_2_camera.PNG`).

**iPad (시뮬레이터 or 실기기)**
3. iPad Pro 13" 시뮬레이터 (Xcode > Window > Devices and Simulators > iPad Pro 13-inch) 에서 같은 화면을 캡처 → `raw/` 에 `ipad_N_이름.PNG`.

## 추천 장면 6종

| # | 장면 | 키 메시지 (스크린샷 위에 얹을 카피) |
|---|---|---|
| 1 | 홈 타임라인 (오늘 섭취 + 리스트) | **"오늘 먹은 거, 사진 한 장이면 끝"** |
| 2 | 카메라로 음식 찍는 순간 (or 분석 로딩) | **"찍기만 하면 AI가 분석해"** |
| 3 | 기록 상세 (칼로리 + 탄단지 + 품목) | **"탄단지까지 자동 분해"** |
| 4 | **인사이트 > 체중 탭 (예측 차트 + 🎯 verdict)** | **"이대로 가면 도달할까? AI가 예측"** ← 메인 임팩트 |
| 5 | 인사이트 > 영양 탭 (주간 추이 + 푸디 코치) | **"푸디가 매일 코치해줘"** |
| 6 | 홈 위젯 (잠금화면 + 홈화면 위젯 실제) | **"홈 위젯으로 남은 칼로리 한눈에"** |

> 4번이 이 앱의 차별 포인트야. 체중 추이 + 점선 예측 + verdict 카드가 한 화면에 보이게 캡처.
> 체중 로그 5–6개 정도 가짜 입력해두고 찍으면 그래프가 풍부하게 나와.

## Claude 가 편집하는 과정

1. 사용자가 `raw/` 에 원본을 채움
2. Claude 가 각 기기군 폴더에 1/2/3/4/5/6 순서로 "스크린샷 + 상단 카피" 합성본을 만듦
3. App Store Connect 업로드는 사용자가 직접

## Apple 가이드라인 주의

- 스크린샷 위에 얹는 텍스트는 **실제 앱 기능** 과 일치해야 함 (허위 광고 거절 사유).
- 가격·할인·경쟁사 비교 문구 금지.
- 디바이스 프레임(베젤) 은 있어도 없어도 OK. 2024년부터 권장은 "베젤 없음" (화면이 screenshot 영역 전체를 채움).
